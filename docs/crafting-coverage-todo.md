# Crafting Coverage TODO

## 方針

クラフト対応は 3 層に分ける。

1. 音声/辞書で対象を拾う
2. 曖昧な派生を deterministic に具体レシピへ解決する
3. Executor が survival UI 操作で実行する

LLM は対象解釈まで。レシピ選択、材料不足、中間クラフト、Action DSL は deterministic resolver と Safety Filter を通す。

## 残量目安

- Agent resolver: survival-critical and common variant families are covered.
- Executor RecipeBook path: runtime RecipeBook entries are scanned as `network_recipe_id` to output item mappings, and `craft` prefers quick craft when the entry is craftable.
- Executor bundled recipe path: the MOD includes vanilla shaped/shapeless crafting recipe patterns, so supported `craft` actions can still run when the RecipeBook entry is locked or absent.
- Executor stonecutting path: the MOD includes vanilla stonecutting recipes, can open/place `minecraft:stonecutter`, and can execute matching stonecutter outputs through survival UI slots.
- Executor smithing transform path: the MOD includes vanilla smithing transform recipes, can open/place `minecraft:smithing_table`, and can execute netherite upgrade outputs through survival UI slots.
- Executor armor trim path: the MOD includes vanilla smithing trim recipes, can execute `smithing_trim` through smithing table slots, and confirms success by `minecraft:trim` component delta plus inventory item+component match instead of output item-count delta.
- Executor special component crafting path: `component_craft` supports dyed leather armor, banner duplicate, map cloning/extending, firework rocket/star/fade through allowlisted component-delta checks.
- Executor direct layout fallback: P0 resolver/executor parity is complete for the covered common survival, bamboo, color, stone, and books/map utility families.
- Full vanilla execution claim: shaped/shapeless crafting, stonecutting, smithing transform, armor trim, and selected special component crafting can run without RecipeBook unlocks, but written book cloning, repair item, shield decoration, tipped arrows, and remaining component-output recipes still need executor-specific flows.
