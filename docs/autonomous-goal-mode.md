# Autonomous Goal Mode

KoeCraft の長期目標は、音声を細かいキー操作ではなく「方針決め」として使い、Executor が周囲 scan、短期計画、復旧、次の小目標選択を自律的に回す体験へ寄せること。

## 体験方針

- ユーザーは「村を探して」「石のツルハシ作って」「エンドラ倒したい」のように目的を言う。
- Agent は現在地、インベントリ、周囲の地形、危険、進捗を見ながら次の 1-3 分の小目標を選ぶ。
- ユーザーの音声は「止まって」「もっと安全に」「急いで」「こっち」「それでいい」のような割り込み・方針変更として常に受け付ける。
- blocked 時は三人称リアクションと3択吹き出しで、次に言える選択肢を出す。
- Rare/progression item は自動補充しない。通常素材や作業台配置、拾得、通常ブロック破壊は bounded な world assist として trace に残す。

## エンドラ討伐のマイルストーン

1. 初期安全化: 食料、木材、石ツール、盾、ベッドまたは簡易シェルター。
2. 鉄準備: 鉄ツール、バケツ、防具、火打ち石。
3. ネザー突入: 溶岩キャストまたは黒曜石フレームでポータル作成。
4. ネザー探索: 砦/要塞ヒント探索、ブレイズロッド入手、安全退避。
5. エンダーパール: エンダーマン戦または村/交易などの補助ルート。
6. 要塞探索: エンダーアイで方向決定、長距離移動、危険時のシェルター。
7. エンド突入: 装備/食料/ブロック確認、ポータル起動。
8. エンド戦闘: 黒曜石柱クリスタル処理、落下/奈落回避、回復、ドラゴン攻撃。

## 実装ステップ

- P0: 作業台、採掘、拾得、移動で詰まった時の復旧を安定させる。
- P1: `active_goal` と `milestone` を Executor に持たせ、1 Action 失敗で goal 全体を捨てない。
- P2: `goal_tick` で再 scan し、完了条件・危険条件・次アクション候補を更新する。
- P3: ユーザーの割り込み音声で goal policy を更新する。
- P4: エンドラ討伐用の milestone planner を deterministic に追加する。

## 3択吹き出しの例

- クラフト詰まり: `作業台を近くに置いて` / `右にずれて` / `一回やめて`
- 採掘詰まり: `手前を掘って` / `階段掘りして` / `一回やめて`
- 移動/探索詰まり: `右によけて` / `橋をかけて` / `一回やめて`
- 拾得詰まり: `アイテム拾って` / `近くを探して` / `一回やめて`
